# Caso de Prueba: TC-01-08 — El login registra evento en bitácora

| Campo | Valor |
|---|---|
| **Rol(es)** | Administrador, Coordinador, Operativo, Usuario |
| **Categoría** | 01 — Autenticación |
| **Metodología** | Login — Bitácora (verificar BD) |
| **Fecha de ejecución** | 2026-05-28 |
| **Motor** | Playwright MCP (Claude Code) + verificación ORM |
| **Estado** | ✅ PASS |

## Descripción
Verifica que un login exitoso crea un registro en `BitacoraEvento` con `tipo='login'`, descripción con el username e IP del cliente (además del `hash_registro` de la cadena de auditoría).

## Precondiciones
- Usuario `admin_prod` / `adminprod`.
- Servidor en `http://127.0.0.1:8000`.

## Pasos ejecutados
| # | Acción | Ubicación / Selector / Dato | Resultado esperado | Evidencia |
|---|---|---|---|---|
| 1 | Login exitoso | `/usuarios/login/` · `admin_prod` / `adminprod` | Redirect al Dashboard | `TC-01-08_paso-1.png` |
| 2 | Consultar bitácora | `manage.py shell` → `BitacoraEvento(tipo='login').last()` | Registro con usuario, descripción, IP y hash | (salida de consola, abajo) |

## Resultado esperado
- Tras el login, existe un `BitacoraEvento` con `tipo='login'`, `descripcion="Inicio de sesión de admin_prod"`, `ip` del cliente, y `hash_registro` encadenado (SHA-256).

## Resultado obtenido
- ✅ Login redirigió al Dashboard.
- ✅ Registro de bitácora confirmado vía ORM (ID 424).

## Verificación en BD
```python
BitacoraEvento.objects.filter(tipo='login').order_by('-id').first()
```
Resultado:
```
ID: 424 | tipo: login | usuario: admin_prod | desc: Inicio de sesión de admin_prod | ip: 127.0.0.1 | fecha: 2026-05-29 05:33:24 UTC
hash_registro: c560a69f1af397c9deae91dda7745560985302cb8e76ca47e5b6b3adb0412c5e
```
